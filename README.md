# Mini Project: Банківський симулятор

## Мета

Створи простий вебзастосунок **«Банківський симулятор»** з використанням HTML, CSS та JavaScript.
Основний фокус — логіка на JavaScript, управління станом, робота з DOM та `localStorage`.

---

## Вимоги до застосунку

Застосунок має дозволяти користувачу:

1. Переглядати поточний баланс.
2. Вносити кошти на рахунок.
3. Знімати кошти з рахунку.
4. Переглядати історію транзакцій.
5. Зберігати дані після перезавантаження сторінки за допомогою `localStorage`.

---

## Структура даних

Стан застосунку має зберігатись у такій структурі:

```js
account = {
  balance: number,
  transactions: [
    { id: string, type: "deposit" | "withdraw", amount: number, date: string }
  ]
}
```

---

## Правила валідації

1. Сума має бути числом, більшим за 0.
2. Заборонено знімати кошти, якщо сума перевищує поточний баланс.
3. Після кожної транзакції потрібно оновити баланс та історію.
4. Після кожної транзакції потрібно зберегти стан у `localStorage`.
5. Під час завантаження сторінки потрібно відновити стан з `localStorage`.

---

## Структура інтерфейсу

Сторінка має містити:

* Блок відображення балансу.
* Поле вводу та кнопку для внесення коштів.
* Поле вводу та кнопку для зняття коштів.
* Список історії транзакцій.
* Опціонально — кнопку очищення історії.

---

## Чеклист для тестування

* Додати внесення коштів і перевірити, що баланс збільшився.
* Додати зняття коштів і перевірити, що баланс зменшився.
* Перезавантажити сторінку і перевірити, що дані збереглись.
* Спробувати зняти більше, ніж доступний баланс, і перевірити обробку помилки.

---

## Додаткові можливості (необов’язково)

* Показати загальну суму внесень і зняттів за допомогою `reduce()`.
* Додати кнопки фільтрації: **Усі**, **Внесення**, **Зняття**.
* Додати форматування дати через `toLocaleString()`.
* Додати кнопку повного очищення всіх даних.
